## 第一部分：蒙格的核心思維模型

在使用任何工具之前，先理解工具背後的哲學。蒙格只用幾個關鍵思維，卻比多數人更少犯錯。

### 思維一：反向思考（Invert, Always Invert）

> *"告訴我我會死在哪裡，我就不去那裡。"*

**一般人問的問題**：這次信號能賺多少？  
**蒙格問的問題**：什麼情況下這個信號必然虧錢？

**應用到本系統**：

在決定是否跟隨信號前，先問：
- 現在是 BEAR 市場嗎？（歷史勝率 43.9%，負期望值）
- 現在是 RECOVERY + RSI < 25 嗎？（最大虧損來源）
- 這個信號有市場背景脈絡嗎？（無 context 勝率只有 30%）

如果三個問題有任何一個答案是「是」，**先停手**，不是先想進場。

---

### 思維二：能力圈（Circle of Competence）

> *"投資者應該只在自己真正了解的領域內行動。"*

本系統目前**有真實邊際優勢**的情境（根據 574 筆歷史數據）：

| 情境 | 勝率 | 平均獲利 | 信號數 |
|---|---|---|---|
| RECOVERY 市場 | 53.3% | +0.201% | 261 |
| RECOVERY + SELL 信號 | ~55% | +0.22% | ~170 |
| RSI < 30 區間 | 51.0% | +0.137% | 143 |
| RSI < 20 區間 | 49.4% | +0.128% | 85 |

**本系統目前沒有優勢**的情境（應該迴避）：

| 情境 | 勝率 | 平均損失 |
|---|---|---|
| BEAR + FLAT + 任何季節 | 18~41% | -0.09% ~ -0.34% |
| BUY 信號（全期） | 41.3% | 負期望 |
| RSI 60-70 的 BUY | 30.6% | -0.225% |
| 無市場背景的信號 | 30.3% | -0.342% |

**能力圈原則**：只在系統有優勢的情境裡行動，在圈外乾坐。大多數時候最好的操作是什麼都不做。

---

### 思維三：雙層過濾（Two-Stage Filter）

蒙格評估投資時有兩道防線：

1. **先問「這能讓我輸嗎？」** — 排除壞的
2. **再問「這真的是好的嗎？」** — 確認好的

**本系統的實現**：

```
原始信號
  ↓
[第一道：環境封鎖] — BEAR|MIXED|FLAT / BEAR|ALT_SEASON|FLAT / BEAR|BTC_SEASON|FLAT？
  ↓ 通過
[第二道：矛盾濾網] — RECOVERY + RSI < 25 + SELL？
  ↓ 通過
[第三道：背景確認] — 有市場 Context 嗎？
  ↓ 通過
值得考慮的信號
```

這不是讓機器人幫你賺錢，而是讓機器人幫你**不做蠢事**。

### 思維四：賠率思維（Odds Thinking）

> *"只有少數幾個好機會，重要的是當機會出現時你要知道，然後下注。"*

當前系統的整體賠率結構：
- 整體勝率：47.4%（略低於 50%）
- 平均獲利：+0.72%，平均虧損：-0.57%
- **賠率比：1.26:1**（正期望值，但很薄）

這意味著：
- 每 100 個信號，期望獲利約 +5.6%（累計）
- 交易成本（手續費 + 滑點）大約 0.1-0.2% 一筆，100 筆就是 10-20%
- **結論：信號越多越虧，只跟最確定的信號**

蒙格說：「我一年只需要幾個好決策。」

## 第二部分：實際操作指南

### 每次收到信號時的 SOP

**Step 1：看市場 Phase（最重要的一步）**

```
工具：Telegram 報告的「大盤環境」區塊
看：Phase 是什麼？
```

| Phase | 代表意義 | 操作原則 |
|---|---|---|
| 🌱 RECOVERY（底部復甦） | 市場剛從低點回升 | **主戰場**，SELL 信號更可靠 |
| 🐂 BULL_RUN（牛市加速） | 強勢上漲中 | BUY 信號較可靠 |
| 📦 DISTRIBUTION（分配頂部） | 主力出貨 | 偏向 SELL，小心 |
| 🐻 BEAR（熊市下跌） | 系統性下跌 | **縮手**，只看不動 |

> **蒙格原則**：在 BEAR 市場裡，每個 BUY 信號都是在跟趨勢作對。
> 歷史數據告訴你，這 278 次裡 56% 都是錯的。

---

**Step 2：看 Season（主導權在誰）**

```
看：Season 是 BTC_SEASON / ALT_SEASON / MIXED？
```

| Season + Phase 組合 | 歷史勝率 | 建議 |
|---|---|---|
| RECOVERY + MIXED | 53% | 正常操作 |
| RECOVERY + BTC_SEASON | 60% | 偏向積極 |
| BEAR + MIXED + FLAT | **41%** ← 已封鎖 | 系統自動否決 |
| BEAR + ALT_SEASON + FLAT | **38%** ← 已封鎖 | 系統自動否決 |
| BEAR + BTC_SEASON + FLAT | **19%** ← 已封鎖 | 系統自動否決 |

> 如果你看到信號備註裡有 `[SOL:blocked]`，就是系統已經代替你做了蒙格會做的決定。

---

**Step 3：看 RSI（判斷信號是否矛盾）**

```
看：RSI 數值是多少？
```

**黃金規則（已寫入代碼）**：
- RSI < 25 + RECOVERY + SELL = **禁止** — 極度超賣卻在復甦期間賣出，這是最大虧損的來源
- RSI 60-70 + BUY = **謹慎** — 歷史勝率只有 30.6%，信號強度弱

**RSI 的正確解讀方式**：

RSI 是「疲勞程度」，不是「方向」。
- RSI < 30 ≠ 一定要賣，可能是最後一跌
- RSI > 70 ≠ 一定要買，可能是最後一漲
- RSI 要配合 Phase 來讀：
  - RECOVERY + RSI < 30 → 超賣反彈，**等 SELL 信號**（空手者等回調買入）
  - BEAR + RSI > 70 → 死貓反彈，**不追**

---

**Step 4：看信號方向（BUY vs SELL 的歷史不對稱）**

```
BUY 勝率：41.3%（比丟銅板差）
SELL 勝率：51.5%（稍微可靠）
```

本系統**做空（SELL）比做多（BUY）可靠**。這符合 RECOVERY 市場的邏輯——市場在底部整理，短線賣出信號往往是在捕捉局部高點的回調。

**對於 BUY 信號，要多一個確認**：
- 是否同時有 1d 日線也看多？（`短多長多 🟢🟢` 組合）
- 是否在 RECOVERY 階段？
- RSI 是否在 30-60 之間（不超買、有動能）？

三個都是 Yes → 這才是值得考慮的 BUY 信號。

### 多面向看板的正確使用方式

系統的多面向看板（v0.6.4）呈現了 5 個獨立維度。蒙格的用法：

```
📈 動能（RSI + MACD）     → 問：現在有力氣嗎？
📐 趨勢（SMA 線位）        → 問：方向確認了嗎？
📊 波動（Bollinger）       → 問：進場時機好嗎？
😱 情緒（F&G + 費率）      → 問：市場情緒理性嗎？
🌐 大盤（Phase/Season）    → 問：環境支持這個方向嗎？
```

**蒙格只需要 3/5 個維度確認，就可以做決定。**  
如果 5 個維度都說同一件事：那幾乎是確定性。  
如果 3 個說 BUY、2 個說 SELL：這是「普通機會」，小注或不動。  
如果維度互相矛盾：**不動**，等待清晰度。

> *"當我無法決定時，答案通常是不。"* — 蒙格

## 第三部分：今日修正的深層原因

### 為什麼 BEAR + FLAT 環境應該封鎖？

**數據說話**：

```
BEAR|MIXED|FLAT：156 筆信號，勝率 41%，avg -0.088%
預期損失：156 × (-0.088%) = -13.7%（累計）
```

這不是「稍微差一點」，這是在系統性地把錢送出去。156 筆信號裡，92 筆是錯的，平均每筆虧損都比獲利多。

**蒙格的邏輯**：「如果一個工廠每週都虧損，你不應該繼續生產，等待奇蹟。你應該停工。」

市場 Phase 的判斷機制（`market_context.py`）偵測的是 BTC SMA50 趨勢。BEAR + FLAT 意味著：BTC 在 SMA50 以下，但短期沒有明顯方向。這個狀態最危險——既不是超跌反彈（可以做多），也不是明確趨勢（可以跟隨）。**不確定性最大的時候，最多只能做壁上觀。**

---

### 為什麼 RECOVERY + RSI < 25 不能 SELL？

這是今日發現的**最反直覺的錯誤**。

```
2026-02-06 00:00 — SOL/USDT  — SELL — RSI=11.8 — RECOVERY → pnl = -5.42%
2026-02-06 00:00 — DOGE/USDT — SELL — RSI=12.9 — RECOVERY → pnl = -3.63%
2026-02-06 00:00 — ADA/USDT  — SELL — RSI=17.5 — RECOVERY → pnl = -3.32%
```

系統看到 RSI=11.8（極度超跌），根據 MACD 或 SMA 邏輯產生了 SELL 信號。  
但在 RECOVERY 市場，RSI 極度超賣是**反彈即將發生的信號，不是繼續下跌的信號**。

這是邏輯矛盾：技術指標說「市場疲憊到極限」，背景環境說「市場正在復甦」。  
兩個訊號互相衝突時，**應該相信環境，不相信短期指標**。

蒙格說：*「如果論點在邏輯上有矛盾，那它一定有一個地方是錯的。」*

---

### 為什麼無背景信號不記錄？

```
空背景（33 筆）：勝率 30.3%，avg -0.342%
這是所有類別裡最差的績效
```

沒有市場背景的信號，相當於在不知道天氣的情況下決定穿什麼衣服。  
技術指標告訴你「現在 RSI=65」，但你不知道這是牛市的 65 還是熊市的 65。  
在牛市 RSI=65 是動能強勁，在熊市 RSI=65 是死貓反彈的高點。

**相同的數字，在不同的情境裡意義完全相反。**

## 第四部分：系統的邊界（蒙格最重視的事）

### 這個系統能做什麼

- ✅ 偵測短線（1h）技術面的超賣/超買訊號
- ✅ 整合市場背景過濾低品質信號（SOL 模組）
- ✅ 記錄歷史績效，讓數據說話
- ✅ 提供多維度資訊，讓你有比盲目猜測更好的基礎

### 這個系統不能做什麼

- ❌ 預測市場方向（沒有人能）
- ❌ 在 BEAR 市場裡穩定獲利
- ❌ 告訴你何時是絕對底部或頂部
- ❌ 替代你自己的判斷

> *"有一些問題，即使擁有最好的思維工具，答案依然是不確定的。接受這種不確定性，是智慧的一部分。"* — 蒙格

### 何時應該完全忽略系統信號

蒙格有一個規則：如果你需要一台計算機才能算出某件事是否值得，那它就不值得。

以下情況，不管系統說什麼，**直接觀望**：

1. 全球宏觀事件正在發生（Fed 會議、地緣政治危機）
2. BTC 在過去 24 小時波動超過 8%
3. 恐懼貪婪指數 < 15（極度恐懼）或 > 85（極度貪婪）
4. 你自己對市場沒有清晰的看法

## 附錄：蒙格的 12 個反向思維清單（使用工具前自問）

開倉前，快速過一遍：

1. **[ ]** 現在是 BEAR 市場嗎？→ 如果是，你為什麼要做多？
2. **[ ]** 信號有 `[SOL:blocked]` 標記嗎？→ 系統已否決，別去覆蓋它
3. **[ ]** RSI < 25 + RECOVERY + SELL？→ 絕對不做，這是最大虧損來源
4. **[ ]** 這是 BUY 信號嗎？→ 歷史勝率 41.3%，需要更多確認才動
5. **[ ]** 信號有市場背景（Phase 不為空）嗎？→ 沒有背景就沒有意義
6. **[ ]** 短線和長線方向一致嗎（`短多長多`）？→ 方向分歧時降低倉位或不動
7. **[ ]** 1h 信號是否和 1d 趨勢衝突？→ 衝突時聽日線
8. **[ ]** 你知道如果錯了，損失是多少嗎？→ 不知道就不動
9. **[ ]** 你是因為「怕錯過機會」才進場嗎？→ FOMO 是虧損之母
10. **[ ]** 這個機會在能力圈內嗎（RECOVERY 的 SELL）？→ 在圈外的別碰
11. **[ ]** 你今天已經做了幾個決策？→ 多於 3 個通常是過度交易
12. **[ ]** 如果系統今天關掉，你還會做這筆嗎？→ 不會就不做
